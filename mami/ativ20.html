Nessa tarefa, temos que, empregando coordenadas polares, fazer e explicar como é possível simular um harmonógrafo simples, utilizando variáveis à sua escolha. Coordenadas polares são coordenadas que não tem como parâmetro os eixos x e y, mas sim, a distância que o ponto está do centro do plano cartesiano (0,0) e o seu ângulo de inclinação. Geralmente utilizam-se as notações r para raio e θ (theta, letra grega) ou a para o ângulo. Para a aplicação que queremos fazer, usar coordenadas polares seria muito útil, porém, infelizmente, o Processing não tem uma conversão formal para este tipo de coordenada. Para a termos, criamos funções que transformam coordenadas cartesianas em polares: A aplicação que queremos fazer é um harmonógrafo, que consiste em um aparato que faz desenhos baseados em círculos com diferentes raios e períodos. Primeiro, criamos as funções que trocam o tipo de coordenada: float p2cx(int r, float a){ return r*cos(a); } float p2cy(int r, float a){ return r*sin(a); } Tendo isso, usamos para desenhar um círculo: void draw(){ x = p2cx(r,a) + width/2; y = p2cy(r,a) + height/2; fill(0); point(x,y); a += PI/180; } Agora que temos o círculo, precisamos fazer formas mais complexas. Para isso, imaginamos que temos um harmonógrafo com uma manivela apenas. Agora, imagine que temos uma manivela no final do braço da primeira, e que, ao invés de desenharmos desta, esta apenas posiciona a segunda e é esta que fará o desenho. e para deixar mais interessante, deixamos as manivelas com raios e períodos diferentes: int r1 = 100, r2 = 50; // raios diferentes. float a1 = 2*PI, a2 = PI/2; float x1, y1, x2, y2; void draw(){ x2 = p2cx(r2,a2) + width/2; y2 = p2cy(r2,a2) + height/2; // usamos a posição da manivela anterior como referencia, ao invés do centro da tela. x1 = p2cx(r1,a1) + x2; y1 = p2cy(r1,a1) + x1; fill(0); point(x1,y1); a1 += PI/180; // períodos diferentes. a2 += PI/170; } Agora, o harmonógrafo funcionando:
